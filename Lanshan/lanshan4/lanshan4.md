### Level0：说一下网速MB和Mb的区别，存储空间的MB和网速的MB的区别

MB全称是MB/s，其中B指**Byte（字节）**；Mb全称是Mb/s，其中b指**bit（比特/位）**

单位换算：1 Byte = 8 bits，1 MB/s = 8 Mb/s

储存空间的MB遵循的进制不同，Windows系统下为**1024**进制，硬盘厂商通常为**1000**进制

二进制：1MB = 1024KB = 1,048,576 字节（Windows 系统）

十进制：1MB = 1000KB = 1,000,000 字节（硬盘厂商）

（话说网速MB常指速度，存储空间的MB常指空间来着，也算是一种不同？）

### Level1：讲一下B/S架构和C/S架构的区别和联系

C/S(Client/Server)架构，即**客户端/服务器**架构需要安装专用客户端软件；而B/S(Browser/Server)架构，即**浏览器/服务器**
架构可以通过浏览器访问，无需安装客户端。往往C/S功能强大但较为笨重，B/S则较为轻便灵活，适合大众应用

一般来说高性能要求，硬件交互等场景常用C/S架构；而互联网应用，信息展示等则常用B/S架构

话说本质上B/S就是C/S的**特殊形式**，B/S中浏览器就是标准化的客户端吧

#### 子网掩码 (Subnet Mask)：

作用： 区分IP地址中哪部分是网络位，哪部分是主机位，必须是连续的1，后面跟连续的0

格式：连续的1表示网络位，连续的0表示主机位

其中/数字的形式则是**CIDR表示法**，其中/n表示有n个连续的1，$32-n$则为连续的0的个数

```text
以/28为例：
11111111.11111111.11111111.11110000
└────────网络位(28位)─────┘ └主机位┘
```

那么`/28` 就表示子网掩码有 28个连续的1，转换为十进制就是255.255.255.240

我们常说的ABC类网络就可以通过子网掩码识别：

```text
/8  = 255.0.0.0          (A类网络)
/16 = 255.255.0.0        (B类网络)
/24 = 255.255.255.0      (C类网络，最常见)
```

查询得最后一段的常见值：

```text
/24 → 0        → 255.255.255.0
/25 → 128      → 255.255.255.128
/26 → 192      → 255.255.255.192
/27 → 224      → 255.255.255.224
/28 → 240      → 255.255.255.240
/29 → 248      → 255.255.255.248
/30 → 252      → 255.255.255.252
```

#### 网络地址 (Network Address)：

网络地址是子网的标识符，代表整个子网，主机位全为0

作用为标识一个子网，不能分配给主机使用，用于路由表

网络地址的计算原理：使用按位与（AND）运算IP地址和子网掩码

```text
IP地址：    192.168.15.90
二进制：    11000000.10101000.00001111.01011010

子网掩码：  255.255.255.240
二进制：    11111111.11111111.11111111.11110000

按位与：
第一段：11000000 AND 11111111 = 11000000 = 192
第二段：10101000 AND 11111111 = 10101000 = 168
第三段：00001111 AND 11111111 = 00001111 = 15
第四段：01011010 AND 11110000 = 01010000 = 80

故网络地址为：192.168.15.80
```

查询得快速算法（不用二进制）：

```text
1. 找到最后一个不是255的掩码段（240）
2. 计算块大小
	256 - 240 = 16
3. 用IP对应段(90)除以块大小
	90 ÷ 16 = 5 余 10
4. 网络地址 = 商 × 块大小 = 5 × 16 = 80

故完整网络地址：192.168.15.80
```

#### 广播地址 (Broadcast Address)：

广播地址是子网内的公共广播系统，用于向所有设备发送消息，主机位全为1

作用为向子网内所有主机发送数据，不能分配给主机使用

计算方法：将网络地址主机位全置1即可

```text
网络地址：  192.168.15.80
二进制：    11000000.10101000.00001111.01010000
                                      └主机位─┘

主机位全置1：
            11000000.10101000.00001111.01011111
                                       ^^^^^^^^
                                       全部变1
可得广播地址：  192.168.15.95
```

#### 可用IP地址：

网络地址和广播地址之间的所有地址，分配给网络设备（计算机、路由器、服务器等）

可用IP数量计算公式：$可用主机数 = 2^{主机位数}-2$

此处可用数 = 16 - 2 = 14个，即192.168.15.81 ~ 192.168.15.94
